# Gene Type Serialization Design

## Overview

Type descriptors should be stored within modules using a module-scoped type registry.

## Design

### Module Type Registry

Each module maintains a mapping of `type_id` (0, 1, 2...) → type descriptors.

### Type References

A type reference consists of two parts:
- **`module_path`**: identifies which module owns the type
- **`type_id`**: integer index into that module's type registry

Special cases:
- **Local types**: `module_path` is empty (or nil)
- **Built-in types**: `module_path` is a reserved string (e.g. `"stdlib"`)

### Generic/Parameterized Types

Each instantiation gets its own `type_id`. For example:
- `Array[Int]` → `(module_path, 5)`
- `Array[String]` → `(module_path, 6)`

### Type Equivalence

A fast native method compares two types for structural equivalence:
- **Fast path**: same `module_path` + same `type_id` → equal
- **Slow path**: compare structure recursively (base type + type parameters)

This is needed because the same logical type (e.g. `Array[Int]`) may appear in different modules with different `type_id`s.

### Recompilation

Recompiling a module invalidates all `type_id`s in that module. No cross-compilation stability guarantees for now.

## Codex Analysis: Current Codebase Findings

*Generated by Codex (gpt-5.3-codex) after deep analysis of ~15 source files.*

### High Severity Issues

1. **TypeId/descriptor mismatch in pipeline.** Compiler emits type ids from `self.output.type_descriptors` (`compiler/control_flow.nim:275`, `compiler/functions.nim:250`), but the finalize step replaces that table with checker output (`compiler/pipeline.nim:498`, `pipeline.nim:745`). Compiler interning and checker interning are not identical (`types/core/matchers.nim:307` vs `type_checker.nim:420`). This can make ids point to wrong or out-of-range descriptors, which degrades to `Any` (`runtime_types.nim:89`).

2. **Applied type checking is not structural.** `(Array Int)` vs `(Array String)` currently compares only constructor name (`runtime_types.nim:325`), not type args. This directly blocks the design goal of giving each generic instantiation its own `type_id`.

3. **Nominal identity is name-based and CU-local**, not module-scoped (`runtime_types.nim:265`, `vm/core_helpers.nim:14,27`). Two modules defining the same class name can collide.

### Medium Severity Issues

4. **`type_aliases` not serialized in GIR.** They exist in `CompilationUnit` (`type_defs.nim:829`) and are populated (`compiler/operators.nim:999`) but GIR only handles descriptors (`gir.nim:427,498`).

5. **TypeDesc shape can't represent module-scoped references.** `TdkNamed`/`TdkApplied` are string-based (`type_defs.nim:224,227`).

6. **Detached local type tables.** `to_function(node)` can create a local type table (`types/core/functions.nim:17`) that is not module-anchored.

## Proposed Concrete Data Structures (Nim)

### Core Types

```nim
type
  ModulePath* = string
    ## Empty string = local module
    ## "stdlib" = built-in types
    ## Otherwise = canonical module path (e.g. "mylib/collections")

  ModuleTypeId* = object
    ## A globally-unique type reference
    module_path*: ModulePath
    local_id*: TypeId  # Index into the module's type registry

  TypeDescKind* = enum
    TdkPrimitive,    # Int, Float, String, Bool, Nil
    TdkClass,        # User-defined class
    TdkApplied,      # Generic instantiation: Array[Int]
    TdkUnion,        # Int | String
    TdkFunction,     # (Fn [Int] Int)
    TdkAny,          # Untyped / gradual
    TdkAlias         # Type alias reference

  ModuleTypeDesc* = object
    ## A type descriptor owned by a module
    name*: string
    kind*: TypeDescKind
    case kind
    of TdkPrimitive:
      prim_tag*: uint8  # Maps to NaN tag bits for fast checks
    of TdkClass:
      parent*: Option[ModuleTypeId]  # Superclass
      properties*: seq[tuple[name: string, type_ref: ModuleTypeId]]
    of TdkApplied:
      base*: ModuleTypeId            # e.g. Array
      type_args*: seq[ModuleTypeId]  # e.g. [Int]
    of TdkUnion:
      members*: seq[ModuleTypeId]
    of TdkFunction:
      param_types*: seq[ModuleTypeId]
      return_type*: ModuleTypeId
    of TdkAny:
      discard
    of TdkAlias:
      target*: ModuleTypeId
```

### Module Type Registry

```nim
type
  ModuleTypeRegistry* = object
    module_path*: ModulePath
    descriptors*: seq[ModuleTypeDesc]   # Index = local TypeId
    name_index*: Table[string, TypeId]  # Fast lookup by name
    version*: uint32                    # Incremented on recompile

  GlobalTypeRegistry* = object
    modules*: Table[ModulePath, ModuleTypeRegistry]
    stdlib*: ModuleTypeRegistry         # Pre-populated with builtins
```

### Type Equivalence

```nim
proc types_equivalent*(a, b: ModuleTypeId, reg: GlobalTypeRegistry): bool =
  ## Fast path: same module + same local_id
  if a.module_path == b.module_path and a.local_id == b.local_id:
    return true
  ## Slow path: structural comparison
  let desc_a = reg.resolve(a)
  let desc_b = reg.resolve(b)
  return structural_eq(desc_a, desc_b, reg)

proc structural_eq*(a, b: ModuleTypeDesc, reg: GlobalTypeRegistry): bool =
  if a.kind != b.kind: return false
  case a.kind
  of TdkPrimitive: return a.prim_tag == b.prim_tag
  of TdkApplied:
    return types_equivalent(a.base, b.base, reg) and
           a.type_args.len == b.type_args.len and
           zip(a.type_args, b.type_args).allIt(types_equivalent(it[0], it[1], reg))
  of TdkUnion:
    # Set equality — every member in a has an equivalent in b and vice versa
    return set_equivalent(a.members, b.members, reg)
  of TdkFunction:
    return a.param_types.len == b.param_types.len and
           zip(a.param_types, b.param_types).allIt(types_equivalent(it[0], it[1], reg)) and
           types_equivalent(a.return_type, b.return_type, reg)
  of TdkClass:
    # Nominal — must be same module + id (already failed fast path)
    return false
  of TdkAny: return true
  of TdkAlias:
    return types_equivalent(a.target, b.target, reg)
```

## GIR Serialization Changes

Current GIR (v13) serializes `type_descriptors: seq[TypeDesc]` per CU.

**Proposed GIR v14:**
```
type_registry:
  module_path: string
  version: uint32
  descriptors: seq[ModuleTypeDesc]    # Each with ModuleTypeId references
  aliases: seq[tuple[name: string, target: ModuleTypeId]]
  imports: seq[ModulePath]            # Modules whose types we reference
```

This replaces the current `type_descriptors` + `type_aliases` split and makes cross-module references explicit.

## Migration Plan

### Phase 1: Unify Interning (Fix High-1)
- Single interning path for type descriptors (eliminate compiler vs checker mismatch)
- Both `matchers.nim` and `type_checker.nim` should use the same `intern_type_desc` function
- **Files:** `pipeline.nim`, `matchers.nim`, `type_checker.nim`

### Phase 2: Add Module Path to TypeDesc
- Extend `TypeDesc` with `module_path` field (or wrap in `ModuleTypeDesc`)
- Populate `module_path` from `CompilationUnit.name` / source file path
- Built-ins get `"stdlib"`, local types get `""`
- **Files:** `type_defs.nim`, `core_helpers.nim`

### Phase 3: Structural Applied Type Comparison (Fix High-2)
- `runtime_types.nim:325` — compare type args, not just constructor name
- This enables `Array[Int] != Array[String]` at runtime
- **Files:** `runtime_types.nim`

### Phase 4: Module Type Registry
- Create `ModuleTypeRegistry` and `GlobalTypeRegistry` types
- Replace `CompilationUnit.type_descriptors` with module registry
- Update GIR serialization to v14
- **Files:** `type_defs.nim`, `gir.nim`, `pipeline.nim`

### Phase 5: Type Equivalence Native Method
- Expose `types_equivalent` as a Gene built-in (e.g. `(type-eq? type1 type2)`)
- Fast path for same-module same-id, slow path for structural
- **Files:** `runtime_types.nim`, new `type_equiv.nim`

## Risk Checklist

- [ ] Phase 1 may break existing tests that depend on current interning order

- [ ] Phase 4 is the largest change — needs careful migration of all CU consumers
