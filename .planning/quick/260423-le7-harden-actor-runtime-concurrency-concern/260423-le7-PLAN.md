# Quick Task 260423-le7: Harden Actor Runtime Concurrency Concerns

## Scope

Address the concrete actor runtime reliability concerns from review:

1. Replace direct actor/thread message id increments with a shared allocator.
2. Prepare initial actor state using actor send-tier isolation.
3. Reject `VkBlock` actor handlers until they have explicit freeze/sendability semantics.
4. Bound actor-originated parked sends so full mailboxes cannot accumulate unbounded `pending_sends`.

The `__thread_error__` map envelope is intentionally left unchanged per user instruction.

## Tasks

### 1. Runtime Hardening

Files:
- `src/gene/vm/thread.nim`
- `src/gene/vm/thread_native.nim`
- `src/gene/vm/runtime_helpers.nim`
- `src/gene/vm/actor.nim`

Action:
- Add/export a shared message id allocation helper.
- Use it from actor wake, reply, send, shutdown, and internal thread helper paths.
- Clone/share actor spawn state via `prepare_actor_payload_for_send`.
- Permit actor handles in actor transport by cloning the handle id.
- Reject `VkBlock` handlers in `actor_spawn_value` and remove the block execution branch from actor dispatch.
- Add a pending send bound equal to the mailbox limit.

Verify:
- Actor runtime compiles.
- Existing reply and send-tier behavior still passes.

Done:
- Runtime paths no longer mutate `next_message_id` directly from actor/runtime helper send sites.
- Mutable spawn state is isolated from caller mutation.
- Block handlers fail at spawn.
- Actor-originated send parking is bounded.

### 2. Regression Tests

Files:
- `tests/integration/test_actor_runtime.nim`
- `tests/test_phase2_actor_send_tiers.nim`

Action:
- Add an integration test that mutates spawn state after spawn and proves the actor sees the cloned initial state.
- Add a unit test that `VkBlock` handlers are rejected.
- Add a mailbox overflow test that parks one actor-originated send and rejects the next without blocking the worker.

Verify:
- Run focused actor tests.

Done:
