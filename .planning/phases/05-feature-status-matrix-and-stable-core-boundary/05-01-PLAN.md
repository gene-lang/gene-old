<objective>
Publish a truthful public feature-status matrix and stable-core boundary so
users can tell what Gene supports today, what is beta or experimental, what is
future-only, and what has been removed.
</objective>

<threat_model>
Primary risk: documentation overclaims stability and misleads users into
building on incomplete or removed surfaces.

Mitigations:
- Only mark a feature stable when the implementation exists, public docs/specs
  describe it, and focused tests are present or clearly referenced.
- Mark partial surfaces as beta or experimental and list known gaps in the
  matrix instead of hiding them in subsystem docs.
- Keep removed thread-first APIs removed in public docs; describe native
  workers only as internal actor/runtime substrate.
- Keep this phase docs-only. If runtime gaps are found, document the gap and
  defer implementation to Phase 06, Phase 07, Phase 08, or a future phase.
</threat_model>
