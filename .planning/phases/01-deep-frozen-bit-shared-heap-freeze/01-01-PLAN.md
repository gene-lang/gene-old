# Phase 1 / Plan 01-01: Header bits + O(1) accessors

**Goal**: Add `deep_frozen` and `shared` bits to every managed-object header and
expose them as O(1) accessors on `Value`. Substrate only — no semantics change
in this plan. Subsequent plans consume these accessors.

**Locks user decision D-03**: bits live on the managed-object header, not on
the NaN tag.

## Approach

1. Add a `flags: uint8 = 0` field next to the existing `ref_count` / `frozen`
   fields on every managed header type: `Gene`, `String`, `ArrayObj`,
   `MapObj`, `HashMapValue`-equivalent on `Reference` for `VkHashMap`,
   `BytesValue`-equivalent for `VkBytes` (bytes is already NaN-tagged immediate
   for ≤6 bytes, so only the heap path needs the field), `InstanceObj`,
   `Class`, `BoundMethod`, `Function`, `Closure`, `NativeFn`.
2. Introduce module-level bit constants:
   ```nim
   const
     DeepFrozenBit*: uint8 = 0b0000_0001
     SharedBit*:     uint8 = 0b0000_0010
     # bits 2..7 reserved for future phases
   ```
3. Add `proc deep_frozen*(v: Value): bool {.inline.}` and
   `proc shared*(v: Value): bool {.inline.}` that:
   - Return `true` / `false` respectively for non-heap kinds (Int/Float/Bool/
     Nil/Char/Symbol/Keyword and immediate Bytes via `BYTES_TAG`/`BYTES6_TAG`)
     with no allocation.
   - Dispatch on the managed tag and read `hdr.flags and BIT`.
4. Add `proc setDeepFrozen*(v: Value)` / `proc setShared*(v: Value)` helpers
   that bit-set on the managed header (no-op on non-heap kinds for
   `setDeepFrozen`; error on non-heap kinds for `setShared` since a non-heap
   value has no header to share).
5. Do **not** rename the existing shallow `frozen: bool` field per D-13. The
   new deep variant is `deep_frozen` to keep names disambiguated.
