# Phase 1 / Plan 01-05: Shared-heap semantics verification

**Goal**: Close HEAP-01 by proving that a frozen value allocated on the
single unified heap is safely reachable from any thread — i.e. by showing a
regression test where thread A freezes a composite value and thread B
retains, dereferences, and releases it with no data race, no torn RC, no use
after free.

**Locks user decision D-08**: there is no separate shared-heap allocator in
Phase 1. The "shared heap" is semantic (tag-on-heap), not physical.

## Approach

### What this plan is not

- Not adding a new allocator.
- Not changing the `Value` tag space.
- Not introducing any new concurrency primitive.

### What this plan is

1. A doc update in `docs/proposals/actor-design.md` section "B. Shared heap
   allocation" clarifying that Phase 1's realization is tag-on-heap, with a
   forward pointer to the follow-up perf-tuning workstream (deferred from
   D-08) for a dedicated pool allocator.
2. A module-header comment in `src/gene/types/memory.nim` stating the
   shared-heap invariant explicitly:
   - A managed value observable with `shared == true` was published by a
     `freeze_*` operation that set the bit under the same write barrier as
     the publication.
   - Readers on other threads must only dereference values reached through
     `shared == true` pointers OR the same-thread owned heap.
   - Crossing threads with `shared == false` is a bug detectable by the
     retain-branch assertion (a debug-mode check in Plan 01-04 can flag it).
3. A cross-thread stress test. Concrete shape:
   - Thread A builds a deeply nested `(array ... (map ...) (gene ...))`
     value and calls `freeze_value`.
   - Thread A publishes the raw `uint64` via a lock-free slot.
   - N worker threads spin-load the slot, retain, read every field
     recursively, release.
   - Main thread verifies the final refcount equals the expected count and
     the value is still reachable (no use-after-free).
   - Run under `--mm:orc --threads:on` and under `tsan` if the build lane
     supports it; otherwise note the gap.

## Tasks

- **T1**: Write the doc update anchoring Phase 1's tag-on-heap realization
  and the deferred pool-allocator follow-up.
- **T2**: Add the module-header invariant comment in `memory.nim`.
- **T3**: Implement `tests/test_phase1_shared_heap.nim` with the cross-
  thread stress pattern described above. Parameterize thread count
  (default 8), iteration count, and value shape.
- **T4**: Run the stress test 100 iterations locally; record whether any
  run failed and under which mode (plain vs tsan). Record in the SUMMARY.
- **T5**: Phase 0 acceptance sweep (no regression).
