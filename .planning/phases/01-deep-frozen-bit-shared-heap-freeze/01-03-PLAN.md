# Phase 1 / Plan 01-03: Mutation opcode guards for `deep_frozen`

**Goal**: Extend every in-place mutation opcode so a write to a `deep_frozen`
value raises a typed `FrozenWriteError`. The existing shallow `frozen` check
stays exactly as-is; this plan adds the `deep_frozen` arm alongside it. No
changes to instruction encoding.

**Depends on 01-01** for the `deep_frozen` accessor; independent of 01-02
(freeze op creates deep-frozen values, but 01-03's guard is tested directly
by setting the bit manually in a unit test, so plans can land in either
order).

## Approach

### Error type

In `src/gene/types/core/value_ops.nim` add:

```nim
type
  FrozenWriteError* = object of CatchableError
    target_kind*: ValueKind
    op*: string             # e.g. "IkSetChild", "IkArrayPush"
```

A single raiser helper:

```nim
proc raise_frozen_write*(op: string, target: Value) {.noinline, noreturn.} =
  var e = new_exception(FrozenWriteError,
    "cannot mutate deep-frozen " & $target.kind & " via " & op)
  e.target_kind = target.kind
  e.op = op
  raise e
```

### Opcodes to guard

Audit `src/gene/vm/exec.nim` and guard every in-place mutator. Working
inventory (confirm at implementation time by grepping `Ik*` handlers that
write to managed targets):

- Array: `IkArrayPush`, `IkArraySet`, any `IkSetChild` targeting VkArray.
- Map / HashMap: `IkMapPut`, `IkHashMapPut`, any `IkSetMember` / `IkSetProp`.
- Gene: `IkGeneSetType`, `IkGenePropSet`, `IkGeneChildPush`.
- Instance: `IkSetMember` / `IkSetProp` on VkInstance (Instance isn't in the
  freeze MVP scope, but the guard is still cheap and future-proof — the
  guard short-circuits on `!deep_frozen`, so the cost is a predictable
  branch).
- Bytes: `IkBytesSet` / `IkBytesAppend` if present on the heap path.

For each site, insert **before** the write:

```nim
if target.deep_frozen:
  raise_frozen_write("IkArrayPush", target)
```

The shallow `frozen` check already present must stay — Phase 1 does not
change shallow-frozen semantics (D-13).

## Tasks

- **T1**: Add `FrozenWriteError` + `raise_frozen_write` helper.
- **T2**: Guard every in-place mutator identified in the `exec.nim` audit.
  Each guard is one branch; document the audit result in the commit message
  so the SUMMARY can reference it.
- **T3**: Write `tests/test_phase1_frozen_write_guard.nim` that, for each
  guarded opcode: constructs a managed value, sets `deep_frozen` directly
  (no dependency on Plan 01-02), attempts each mutation, asserts
  `FrozenWriteError` with correct `target_kind` and `op`.
- **T4**: Run Phase 0 acceptance sweep to prove non-frozen paths are
  unchanged (zero guard triggers under the existing test corpus).
