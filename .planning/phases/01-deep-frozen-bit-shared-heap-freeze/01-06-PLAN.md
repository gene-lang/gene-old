# Phase 1 / Plan 01-06: "sealed" vs "frozen" naming finalization

**Goal**: Align all user-visible Phase 1 surface area with the final naming
locked in CONTEXT.md decision D-13:

| Concept | Term | Origin |
|---------|------|--------|
| Shallow literal form `#[]` / `#{}` / `#()` | **sealed** | existing `frozen: bool` field (kept on disk; renamed in docs/errors only for Phase 1) |
| Deep recursive `(freeze v)` output | **frozen** | new `deep_frozen: bool` bit, new `(freeze)` stdlib op |

**Depends on 01-02** (so the `(freeze)` entry point exists) and **01-03**
(so error messages exist to align). Independent of 01-04 / 01-05.

**Do NOT rename the on-disk `frozen: bool` field in Phase 1.** That is a
hygiene sweep for Phase 1.x per D-13. Wrongly renaming it now causes churn
across the codebase and risks regressions.

## Approach

### User-facing surfaces to sweep

1. **Error messages**:
   - `FreezeScopeError` message (from 01-02): mention "frozen" (deep),
     clarify that already-sealed (shallow) non-MVP kinds still cannot be
     deep-frozen.
   - `FrozenWriteError` message (from 01-03): say "cannot mutate **frozen**
     %s via %s" \u2014 aligned.
   - Any `shallow frozen` / `deep frozen` strings in existing error paths.
2. **Stdlib names / docstrings**:
   - `freeze` in `src/gene/stdlib/freeze.nim` \u2014 docstring: "Recursively
     deep-freezes a value..." mentioning the sealed vs frozen distinction
     in one sentence.
   - Existing shallow literal constructors `#[]` / `#{}` / `#()` \u2014
     docstrings say "sealed" instead of "frozen" (only docstrings; not
     literal syntax).
3. **Design doc**:
   - `docs/proposals/actor-design.md` \u2014 single glossary pass in the
     naming section so every subsequent `sealed` / `frozen` reference in
     the proposal is internally consistent.
4. **Handbook entry**:
   - Add `docs/handbook/freeze.md` (new file, \~100 lines) explaining the
     two concepts, when each applies, the MVP scope, and the Phase 1.5
     forward reference for closures. Link from the handbook index if one
     exists; otherwise leave unlinked and flag in SUMMARY.

### What this plan is not

- Not renaming the on-disk `frozen: bool` field.
- Not changing any Gene syntax.
- Not changing function signatures beyond docstrings.

## Tasks

- **T1**: Audit every occurrence of the words `frozen`, `freeze`, `sealed`,
  `immutable`, and `read-only` in: stdlib docstrings, error messages, and
  the actor-design proposal. Produce a dump in the commit message.
- **T2**: Normalize to the D-13 terminology in the error/docstring/doc
  surfaces. Leave the on-disk field name alone.
- **T3**: Write `docs/handbook/freeze.md`.
- **T4**: Run Phase 0 sweep plus the Phase 1 regression suite from earlier
  plans to confirm no test relies on the old error wording. Update only
  tests that explicitly assert error strings.
